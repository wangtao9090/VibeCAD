# Round 5 设计：可指代性（标注渲染 + 标签注册表）+ 首批面/边特征工具

**日期**：2026-06-09 ｜ **状态**：设计已批准（用户确认范围与方案）

## 1. 问题与需求评估

当前 `render_part` 输出纯 PNG。用户想"在某个面打孔 / 某条边倒角 / 把某个方向拉长"时，**没有任何精确指代手段**——GUI CAD 靠鼠标拾取，对话式 CAD 只有语言，而裸语言指代几何天然歧义。可指代性是后续一切特征工具的前置依赖，本轮补上。

用户的三个例子对应三种操作类型，指代通道各不相同：

| 例子 | 操作类型 | 指代通道 | 结论 |
|---|---|---|---|
| "在 A 面打一个孔" | 面级特征 | 面标签 A/B/C… | **必标**（默认 annotate="faces"） |
| "C 线倒角" | 边级特征 | 边标签 E1/E2… | **按需标**（全标会爆炸：裸方块 12 条边 + 每孔口 2 圆；要倒角时才出边标注图，可按面过滤） |
| "B 拉长 5 公分" | 参数修改 | **参数名**（非边标签） | 对象是参数化的，改尺寸走参数重算。图上画**尺寸线**（L/W/H 数值），用户直接说"长度改 45"。参数修改工具留 Round 6，本轮先画尺寸线 |
| （点） | — | 语义表达 | **不标**。"A 面中心"、"E3 中点"足够，MVP 无独立点选场景 |

## 2. 方案选型

| 路线 | 判断 |
|---|---|
| **A. matplotlib 标注渲染（选定）** | 在现有 PNG 管线扩展：逐面 tessellate（glTF 路径已有此模式）→ 可见面判定 → 面心贴标签。零新依赖、纯函数可快测 |
| B. FreeCADGui 离屏高亮 | 离屏 GUI 无显示环境可用性未验证，与无头运行时架构冲突，留后续轮 |
| C. glTF + 自研 viewer 真交互拾取 | 终局体验但要做 App，超出 Chat-native 边界。glTF face extras 已铺路，与 A 不冲突，留未来 |

## 3. 设计

### 3.1 工具面（用户可见行为）

```
render_part(view="iso", annotate=None|"faces"|"edges", edges_of=None)
  annotate="faces" → 图：可见面贴 A/B/C… 标签 + 包围盒三向尺寸线（L/W/H）
                   → 表：{"A": "平面 40×30mm，法向+Z（顶面）", "B": "圆柱内壁 ⌀12（孔壁）", …}
                     不可见面也列在表中并注明建议视角；表给 AI 看（用户说"顶面"→AI 译成 A）
  annotate="edges" → 边标 E1/E2… 于边中点；edges_of="A" 只标 A 面的边

add_hole(face, diameter, depth=None, offset=[0,0])
  face="A"；depth=None 通孔；offset 为面内毫米坐标（原点=面心）
fillet_edges(edges, radius)    # 圆角，edges=["E1","E2"]
chamfer_edges(edges, size)     # 倒角
```

特征操作成功后**自动返回新标注图**（几何变了标签即刷新，用户永远看到最新指代）。

### 3.2 标签注册表 + 指纹校验（`engine/naming.py`）——解持久命名问题（MVP 级）

FreeCAD 面/边索引在布尔/重算后会重排——"A 面"打完孔后索引可能指向别的面。若不处理，会倒错面还报成功（"信任了错误的成功信号"的指代版）。方案：

- **快照式标注**：每次出标注图时生成 `标签 → 指纹` 映射存入 Session。指纹：面 =（Surface 类型, Area, CenterOfMass, 面心法向）；边 =（Curve 类型, Length, 中点）。
- **执行前指纹校验**：特征工具拿到 `face="A"` → 按指纹在**当前**几何中容差比对找回该面；对不上（几何已变/多义）→ 抛清晰错误"标签 A 已过期，请重新查看标注图（render_part annotate）"，**绝不静默猜面**。这是"几何断言才可信"纪律在指代层的延伸。
- 引用只对最近一次快照有效；操作成功后自动出新标注图完成标签轮换。

### 3.3 标注绘制（`feedback/annotate.py`，纯函数可快测）

- 逐面 tessellate（复用 glTF 导出器的逐面模式）→ 各面三角集；面间着色微分异 + 面心放标签（matplotlib 3D 文本自动投影，白底小框）。
- **可见性**：面心法向朝向相机（dot < 0）才在图上贴标签，背面不贴防误导；非平面（孔壁等）图上尽力、标签表描述兜底。**不做精确遮挡**（MVP 用朝向近似 + 多视角覆盖）。
- **尺寸线**：包围盒三向 L/W/H（通用，不依赖图元类型）。

### 3.4 特征工具（`tools/features.py`）

- `add_hole`：解析 face → 校验是平面 → 孔轴 = 面法向反向，位置 = 面心 + 面内 offset → 内部复用 cylinder + Part::Cut → **体积严格减少断言**（继承 Round 4 教训）。offset 方向语义（MVP）：取全局坐标轴在该面内的投影——对轴对齐平面即"全局 X/Y（或被法向占用时的下一对轴）"，直观且无需用户理解面参数方向；主用例是 `[0,0]`（正中）。
- `fillet_edges` / `chamfer_edges`：`Part::Fillet` / `Part::Chamfer` 文档对象（保参数化）→ 断言 isValid + **面数增加** + 体积有变化（圆角凸边减料、凹边加料，**不能**断言体积方向）。OCCT 圆角对部分边会失败——断言逮住后报清楚哪条边。

### 3.5 server 集成

新工具 `add_hole` / `fillet_edges` / `chamfer_edges`，`render_part` 加 `annotate`/`edges_of` 参数；全部走 `_runtime_guard` 三态守卫 + 失败转结构化 `{ok:False}`。握手纯净纪律不变（模块级不 import FreeCAD/matplotlib）。

## 4. 测试与验收

- **快测（dev venv）**：指纹比对纯逻辑（容差/多义/不匹配）；可见性判定（已知法向网格）；annotated_png 出合法 PNG（mock 网格）；server 新工具 mock 委托；握手纯净回归。
- **真机慢测**：标注图生成非空；**cut 之后指纹仍指对原面**（持久命名核心回归）；`add_hole("A")` 体积精确断言（V=V₀−πr²h）；fillet 后面数增加 + isValid；标签过期抛错路径。
- **黑盒终验（真实 MCP 协议）**：render 标注图 → "在 A 面正中打 8mm 孔" → 新标注图；**人眼看图确认**孔在指定面 + 标签清晰可读。
- **Spike 先行**：正式动工前用真机管线出一张带孔方块的面标注原型 PNG 人眼检验（标签重叠/字号/孔壁标签位置只有看图才知道），样式不行先调再开工。

## 5. 风险

1. matplotlib 小图（440px）标签可读性 → spike 验证，必要时加大图幅/字号。
2. FastMCP 单工具返回"图 + 标签表"双内容 → 计划阶段验证（FastMCP 支持 list 返回多 content block；不行则图与表拆分返回）。
3. OCCT fillet 脆弱（部分边失败）→ 几何断言兜底 + 错误信息指明边。
4. 非平面标签位置可能悬空（圆柱面质心在轴上）→ 接受，标签表描述兜底。
