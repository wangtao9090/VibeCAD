## 审查实测结论（执行时直接引用，勿再踩）

1. **CLI 2.1.2 的 v0.4 schema：`server` 必填 type/entry_point/**`mcp_config`**（command 必填）**——uv 类型官方写法：`{"command": "uv", "args": ["run", "--directory", "${__dirname}", "<entry>"]}`（宿主把 `uv` 解析为自管二进制）。
2. **`FastMCP("vibecad", version=...)` 抛 TypeError**（mcp 1.27.2 不透传）；可行方案：构造后 `mcp._mcp_server.version = __version__`，实测 `create_initialization_options().server_version` 即返回设定值。
3. **manifest `tools.items` 为 `additionalProperties: false`**——不能塞 annotations；工具行为标注走 server 侧 `ToolAnnotations`（运行时 tools/list 返回）+ 提交表单文字。
4. **`mcpb pack` 默认不排除 .venv/.claude/__pycache__**（实测会打进包，仓库 .venv 162MB）——.mcpbignore 必须显式排除。
5. **manifest `version` 仅 string 无格式约束**——三处统一 `0.2.0rc1`（PEP 440）可过 validate；rc 版本发 PyPI 是预发布（pip/uvx 默认不装），全链路演练安全。
6. **gh release create 不会从 tag 名自动推断 prerelease**——rc 需显式 `--prerelease`，否则顶掉 `releases/latest`（手册主链路就指它）；删 tag 不删 release（`gh release delete` 另跑）。
7. mcp 1.27.2 私有 API 实测存在：`mcp._tool_manager.list_tools()`、`mcp._mcp_server`；`pack <dir> [output]` 自动建输出目录且内置先 validate。

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

jobs:
  pypi:
    runs-on: ubuntu-latest
    environment: pypi
    permissions:
      id-token: write
    steps:
      - uses: actions/checkout@v4
      - name: tag/version consistency guard
        run: |
          TAG_VER="${GITHUB_REF_NAME#v}"
          PY_VER=$(grep -m1 '^version = ' pyproject.toml | cut -d'"' -f2)
          if [ "$TAG_VER" != "$PY_VER" ]; then
            echo "::error::tag $GITHUB_REF_NAME 与 pyproject version $PY_VER 不一致——拒绝发布（防版本号烧毁）"
            exit 1
          fi
      - uses: astral-sh/setup-uv@v6
      - run: uv build
      - uses: pypa/gh-action-pypi-publish@release/v1
  mcpb:
    runs-on: ubuntu-latest
    permissions:
      contents: write   # 创建 GitHub Release
    steps:
      - uses: actions/checkout@v4
      - run: npx -y @anthropic-ai/mcpb@2.1.2 validate manifest.json
      - run: npx -y @anthropic-ai/mcpb@2.1.2 pack . VibeCAD.mcpb
      - name: create GitHub Release with .mcpb asset
        env:
          GH_TOKEN: ${{ github.token }}
        run: |
          EXTRA=""
          case "$GITHUB_REF_NAME" in
            *rc*|*a[0-9]*|*b[0-9]*) EXTRA="--prerelease" ;;   # rc 不顶掉 releases/latest
          esac
          gh release create "${GITHUB_REF_NAME}" VibeCAD.mcpb \
            --title "VibeCAD ${GITHUB_REF_NAME}" \
            --generate-notes $EXTRA
```
